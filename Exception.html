Quản Lý Ngoại Lệ (Exception Handling) trong Java

Ngoại lệ (exception) là các tình huống lỗi phát sinh khi chương trình chạy. Java cung cấp cơ chế quản lý ngoại lệ để bạn xử lý lỗi gọn gàng, bảo vệ dữ liệu và giữ trải nghiệm người dùng mượt mà.

Nội dung
  • 1) Giới thiệu về ngoại lệ trong Java
  • 2) Cấu trúc cơ bản của quản lý ngoại lệ
  • 3) Ví dụ quản lý ngoại lệ
  • 4) Các loại ngoại lệ chính
  • 5) Kết luận

1) Giới thiệu về ngoại lệ trong Java

Trong Java, mọi ngoại lệ đều là đối tượng của Throwable. Hai nhóm chính:

  • Checked exceptions: biên dịch bắt buộc phải xử lý/khai báo (vd. IOException, SQLException).
  • Unchecked exceptions: không bắt buộc xử lý (thường lỗi lập trình: NullPointerException, ArrayIndexOutOfBoundsException).

2) Cấu trúc cơ bản của quản lý ngoại lệ

  • try – catch: bọc đoạn có rủi ro trong try, xử lý trong catch.
  • finally: luôn chạy (kể cả khi có/không có ngoại lệ), dùng để đóng tài nguyên.
  • throw: tự ném một ngoại lệ.
  • throws: khai báo phương thức có thể ném ngoại lệ để caller biết mà xử lý.

3) Ví dụ về quản lý ngoại lệ

Ví dụ 1: try – catch

<!-- ArithmeticException: chia cho 0 -->
public class Example {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("Lỗi: Không thể chia cho 0");
        }
    }
}

Khối catch bắt ngoại lệ và giúp chương trình không bị “văng”.

Ví dụ 2: finally

public class Example {
    public static void main(String[] args) {
        try {
            System.out.println("Thực thi trong try");
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("Lỗi: " + e.getMessage());
        } finally {
            System.out.println("Khối finally luôn được thực thi");
        }
    }
}

finally thường dùng để đóng file/kết nối DB, giải phóng tài nguyên.

Ví dụ 3: throw

public class Example {
    public static void checkAge(int age) {
        if (age < 18) {
            throw new ArithmeticException("Tuổi phải ≥ 18");
        }
        System.out.println("Đủ tuổi");
    }

    public static void main(String[] args) {
        try {
            checkAge(15);
        } catch (ArithmeticException e) {
            System.out.println("Ngoại lệ: " + e.getMessage());
        }
    }
}

Ví dụ 4: throws

import java.io.*;

public class Example {
    public static void readFile() throws IOException {
        FileReader file = new FileReader("nonexistentfile.txt");
        BufferedReader br = new BufferedReader(file);
        br.readLine();
        br.close();
    }

    public static void main(String[] args) {
        try {
            readFile();
        } catch (IOException e) {
            System.out.println("Lỗi đọc file: " + e.getMessage());
        }
    }
}

4) Các loại ngoại lệ chính trong Java

  • Checked: bắt buộc xử lý/khai báo – ví dụ IOException, SQLException.
  • Unchecked: do lỗi logic/chạy – ví dụ NullPointerException, ArithmeticException, ArrayIndexOutOfBoundsException.

5) Kết luận

Thành thạo try-catch, finally, throw, throws giúp bạn kiểm soát lỗi, viết code an toàn và dễ bảo trì. Khi thiết kế API, hãy chọn đúng loại ngoại lệ và thông điệp rõ ràng để caller xử lý chính xác.
